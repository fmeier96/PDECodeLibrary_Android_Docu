Detailed Description

The root animation.

The root animation is a singleton (although you could create several root animations, which would then operate in parallel and have different controlling parameters like timing factors, this is not usually done). The static function members all link to the root singleton.

Listener on global frame timing.

void de.telekom.pde.codelibrary.ui.animation.PDEAnimationRoot.runloopTiming ( )
inline

Delayed timing on runloop.

We don't have to clear the changedRegisteredWithParent flag - either we're not running, then we do change management here. Or it stays, then the next frameloop will do the work. If we get stopped inbetween, the unregister from the frameloop will take care that runloop timing is set again.
